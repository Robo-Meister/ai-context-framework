The framework includes a lightweight networking layer for synchronizing context between agents. It currently consists of the following components:

NetworkManager

Wraps any NetworkInterface implementation and spawns a background thread to deliver incoming messages.

SimpleNetworkMock

An in-memory mock network used for development and unit tests. It queues messages and invokes callbacks from a background thread.

DistributedContextManager

Combines ContextManager with a network backend to broadcast updates and apply changes received from peers.

Example Usage

from caiengine.core.context_manager import ContextManager
from caiengine.network.network_manager import NetworkManager
from caiengine.network.simple_network import SimpleNetworkMock
from caiengine.core.distributed_context_manager import DistributedContextManager

cache = ContextManager()
network = SimpleNetworkMock()
manager = DistributedContextManager(cache, NetworkManager(network))

manager.update_context("task", {"status": "in_progress"})
# ... other nodes receive the update ...

ContextBus

ContextBus mirrors context updates across multiple network backends and can optionally filter updates before relaying them. It enables real-time multi-node cooperation.

NodeRegistry

NodeRegistry is a small utility that records active nodes in Redis. Each node registers its RoboID and network address, making discovery of peers easy. See RoboID Addressing for details on the format and how existing protocols are mapped into CaiEngine.

Agent Mesh Topology

To express relationships between agents the framework ships with AgentNetwork, a light-weight undirected graph that keeps track of neighbour links and their optional weights. It is deliberately minimal so it can run entirely in-memory for orchestration experiments and the automated tests.

What is still missing?

The agent mesh does not by itself move messages between processes. For actual inter-agent communication you need a NetworkInterface implementationâ€”such as SimpleNetworkMock for tests or PubSubNetwork backed by a real CommunicationChannel. This repository now ships ready-made pub/sub channel adapters for Redis and Kafka, but you still need to install the matching dependency and wire the channel into PubSubNetwork, then into NetworkManager or DistributedContextManager so updates flow across nodes.

Production wiring example (Redis)

from caiengine.core.context_manager import ContextManager
from caiengine.core.distributed_context_manager import DistributedContextManager
from caiengine.network import NetworkManager, PubSubNetwork, RedisPubSubChannel

cache = ContextManager()
channel = RedisPubSubChannel(host="localhost", port=6379)
network = PubSubNetwork(channel, node_id="node-1")
manager = DistributedContextManager(cache, NetworkManager(network))

manager.update_context("task", {"status": "queued"})

Production wiring example (Kafka)

from caiengine.core.context_manager import ContextManager
from caiengine.core.distributed_context_manager import DistributedContextManager
from caiengine.network import NetworkManager, PubSubNetwork, KafkaPubSubChannel

cache = ContextManager()
channel = KafkaPubSubChannel(bootstrap_servers="localhost:9092", group_id="mesh")
network = PubSubNetwork(channel, node_id="node-1")
manager = DistributedContextManager(cache, NetworkManager(network))

manager.update_context("task", {"status": "queued"})

Production transport adapters

The production transport adapters are optional to keep the core install light. Install the extra dependencies before using them:

  • pip install caiengine[redis] for RedisPubSubChannel
  • pip install caiengine[kafka] for KafkaPubSubChannel

Once installed, use PubSubNetwork with the adapter to connect the transport to NetworkManager or DistributedContextManager.

Network-Aware Hooks

Hooks can be registered on DistributedContextManager and are triggered whenever context updates occur. These hooks can broadcast new events to the network or invoke external services.
