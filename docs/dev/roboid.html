RoboID provides a unified addressing scheme for the CaiEngine networking layer. It captures an agent's type, role, location and optional instance so that peers can reliably route messages and map existing protocols into the framework's network.

Format

A RoboID is expressed as type.role@place#instance:

  • type â€“ broad category of node (e.g. http, worker).
  • role â€“ functional role within the system.
  • place â€“ physical or logical location.
  • instance â€“ optional identifier when multiple nodes share the same attributes.

Protocol Mapping

Existing services can be bridged into CaiEngine by translating their native addresses into RoboIDs. For example, an HTTP service at http://api.example might register as http.api@cluster-a#1. NodeRegistry stores these RoboIDs along with the original network endpoint so other agents can discover and connect using their preferred protocol.

Example

from caiengine.network.roboid import RoboId

rid = RoboId.parse("service.worker@eu-central#42")
print(rid.node_type)  # service
print(str(rid))       # service.worker@eu-central#42
